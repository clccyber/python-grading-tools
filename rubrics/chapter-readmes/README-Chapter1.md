## Common Chapter 1 Issues

### Syntax Errors
- Missing quotes around strings
- Missing parentheses in `print()` or `input()`
- Typos in function names

### Type Errors
- Forgetting `float()` conversion
- Trying to do math on strings

### Logic Errors
- Wrong operator (+ instead of *)
- Wrong formula
- Missing parts of calculation

### Output Formatting
- Missing units (square units, cubic units)
- No labels on output
- Wrong units (square vs cubic)

---

## Grading Guidelines

### Point Distribution Philosophy

**Chapter 1 focuses on:**
- Does it run? (20-30 points)
- Does it get input? (15-25 points per input)
- Does it calculate correctly? (30-40 points)
- Does it output properly? (10-20 points)

**Be generous in Chapter 1:**
- Students are learning basics
- Syntax errors are expected
- Focus on whether they understand concepts
- Award partial credit liberally

---

## Testing Tips

### Quick Tests

**For all exercises:**
1. Does the file exist with correct name?
2. Does it run without errors?
3. Does it produce output?
4. Is the output roughly correct?

**Detailed checks** (if needed):
- Test with multiple inputs
- Verify calculations manually
- Check output formatting

## Notes for Instructors

### First Programs

Chapter 1 is students' introduction to Python:
- Be patient with syntax errors
- Explain error messages clearly
- Encourage experimentation
- Focus on building confidence

### Emphasis

**What matters:**
- Program runs
- Basic logic is sound
- Output is labeled

**What doesn't matter yet:**
- Perfect formatting
- Optimal solutions
- Advanced techniques

### Feedback

**Good feedback in Chapter 1:**
- "Program works! Good job on the formula."
- "Close! Remember to convert input to float."
- "Output is correct, but add units for clarity."

**Avoid:**
- Overwhelming with advanced concepts
- Being too picky about style
- Expecting professional-level code
